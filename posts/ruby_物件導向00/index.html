    Ruby 物件導向設計實踐 敏捷開發入門

    July 21, 2022

    Practical Object-Oriented Design in Ruby: An Agile Primer#

    本書是 Sandi Metz 大神的著作。 此系列文用來記錄目前的讀書狀況。釐清觀念及深度了解物件導向，學習設計具重複使用、簡單更改特性的物件設計。

    物件導向基礎#

    物件導向有三個特性，目的是為了重複使用，容易修改。

    容易修改指的是：

    • 修改後也不會影響其他功能運作
    • 遇到需求變更也只需要小幅更動

    封裝（Encapsulation）#

    class Greeting
  def say_hello(word)
    puts "#{word}"
  end
end

    封裝是將方法包裝起來，像是介面的概念，如同遙控器裡的按鈕，或是 BMI 計算等功能，將其包裝起來。

    繼承（Inheritance）#

    class People < Greeting

end

    People 繼承至 Greeting，提供一個可操作介面，而非直接觸碰 Greeting 裡面的方法。

    原因在於，直接修改 Greeting 未來新增其他方法後，可能會發生未知的錯誤，並使 Greeting 裡面的東西變得過於龐大，難以修改，也與該物件名稱的所需的行為脫軌。

    多型別（Polymorphism）#

    John = People.new("我就棒 123")
John.say_hello #我就棒 123

Ellen = People.new("我最棒 456")
John.say_hello #我最棒 456

    定義相同的物件，產生出不同的效果（型別），就像 John 是我就棒 123，而 Ellen 是 我最棒 456。

    總結#

    上面舉例的方法或許有點抽象，但以 Rails 的方向去思考，從 ExampleController < ApplicationController 開始，就是在做一個繼承的動作，也因此在 Controller 裡面不需要額外多寫什麼 Action 也能正常作用。
