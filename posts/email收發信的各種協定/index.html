    Email收發信的各種協定

    July 21, 2022

    爲了復刻團隊專案 Gmail，首先需要了解 Email 收發信是怎麼運作，確保自己做的方向正確，一步一步拼湊所需的知識，以及需要準備的哪些東西。

    寄信＆收信#

    SMTP 協定（Simple Mail Transfer Protocol）#

    SMTP 用於驗證 Email 的傳輸。 當使用者寫完信按下送出時，就會透過 SMTP 協定，傳送到 SMTP Server，再發送到收件人的 SMTP Server，等待對方用 POP3 或 IMAP 的方式存取信件。

    SMTP 使用 TCP 協定，確保寄信正常。假如因為某些原因，像是拼錯 Email 信箱，或是沒有這個 Email 信箱，自己就會收到寄信失敗的信件。

    簡單來說 SMTP 就像郵差，負責運送郵件，假如找不到就會退信，如果找到就會放在收信人的信箱裡。

    IMAP （Internet Message Access Protocol）#

    使用 IMAP 的話，資料會保存在 Server 上。當使用者開啟信箱時，會連線到 Server 確認有無新信件，如果使用者建立了一個標籤或是資料夾，Server 也會同步更新，之後傳到每個裝置上，保持一制性。意思是使用者用什麼裝置做什麼，其他裝置也會更著改變。

    簡單來說，IMAP 是讓使用者到伺服器去查看信件，也因此只要有什麼動作發生，就會記錄在伺服器中，當然使用什麼裝置，做什麼變更，都會同步。

    POP3 協定（Post Office Protocol）#

    相較於 IMAP，POP3 單純許多。 只要信件從 Server 進入信箱，就會將備份從 Server 裡刪除。 舉例來說，如果是使用 A 電腦開啟信箱，將信件下載回來，那麼再打開 B 電腦開啟信箱，就不會收到信了，因為信件只會保存在 A 電腦上。

    總結#

    初步簡單判斷，如果是個人信箱，那麼使用 IMAP 會比較方便，而 POP3 比較適合公司使用，讓信件只能保留在公司電腦中。
