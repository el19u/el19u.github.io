Ruby 2.7 warning Using the last argument as keyword parameters is deprecated

Posted on:2022年10月25日 |  at 下午11:29

升級到 Ruby 2.7.X ，專案中碰到的錯誤訊息。

warning: Using the last argument as keyword parameters is deprecated

錯誤訊息要表達的意思是，最後一個引數不能單只有 key 。

更精準的錯誤訊息是 Automatic conversion of keyword arguments and positional arguments is deprecated, and conversion will be removed in Ruby 3. [Feature #14183]

錯誤訊息

因為在 Ruby 3 時， 如果最後一個引數只有 key ，會直接壞掉。

### ruby 3.x.x
  def foo(imString, key: "no key")
    puts "imString: #{imString}"
    puts "key #{key}"
  end

  foo(fooKey: "test") # foo': wrong number of arguments (given 2, expected 1)

Ruby 2.7.x 裡面可以看到，雖然不會壞，但錯誤訊息也提醒，未來將會棄用這個方法。

### ruby 2.7.x
  def foo(imString, key: "no key")
    puts "imString: #{imString}"
    puts "key #{key}"
  end

  foo(fooKey: "test") # warning: Using the last argument as keyword parameters is deprecated
                      # {fooKey: test}
                      # key: no key

要避免錯誤，其實只要將它用{ }包起來就好

  foo({fooKey: "test"}) # {fooKey: "test"}
                        # {key: "no key"}
